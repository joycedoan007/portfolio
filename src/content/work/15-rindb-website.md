## Overview

RinDB is a lightweight, embeddable key-value database built for developers who need fast, low-overhead storage without the complexity of a full database engine. The goal of this project was to design a marketing website that communicates RinDB's technical value clearly to a developer audience — straightforward, no fluff.

The project covered the full design process: empathy mapping, information architecture, sitemap, wireframing, a minimal design system, and high-fidelity UI.

---

## Design Thinking

Before touching pixels, I mapped out who the users are and what they actually need from a landing page.

**Empathy Map** — Developers evaluating lightweight storage solutions want to quickly understand: is this production-ready? How does it compare to alternatives? Can I try it in 5 minutes?

**Problem Scope** — Existing database marketing sites are either too enterprise-heavy (too much selling, not enough signal) or too sparse (open-source projects with zero design). RinDB needed to sit in between: credible, technical, but fast to parse.

**Define** — Core jobs to be done: understand the value prop in under 30 seconds, find integration docs quickly, and build enough confidence to try the CLI.

---

## Design System

**Color palette** — Dark background (`#0D0D0D`) with warm red-orange accents (`#C0392B` → `#E74C3C`) to convey low-level performance and developer culture. High contrast for readability.

**Typography** — System-native sans-serif for body, monospace for code samples. No custom fonts to keep load times minimal — fitting for a performance-focused product.

**Components** — Feature cards with subtle borders, code block snippets, FAQ accordions, and a sticky nav that stays out of the way.

## Final UI

![RinDB Homepage — Desktop](/work/rindb/home-desktop.png)

The hero leads with the product name and a one-line value prop: **"RinDB: A Lightweight, Embeddable Key-Value Database."** A 3D mesh graphic reinforces the technical nature without being decorative noise. Two CTAs — Get Started and View Docs — cover the two primary user intents.

**Why RinDB?** — Three feature pillars: Lightweight & Embeddable, Clean, Safe & Correct, Developer-First.

**How RinDB Works** — Minimal diagram: Minimal Footprint → ETL Tables → Snapshots → Observability. Communicates the architecture in one row.

**Try Without Code** — CLI example block that developers can copy immediately. Reduces friction to first use.
